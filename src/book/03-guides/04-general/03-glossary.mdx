## Internal

[**Calibration Tool**](): NUbots automatic tool for calibrating the cameras in the head which are used for the robot's vision.

[**Green Horizon**](https://nubook.nubots.net/system/subsystems/vision#green-horizon-detector): The edge of the football field, as calculated by the vision system.

[**NUbook**](https://nubook.nubots.net): The NUbots documentation website. Source is hosted on [GitHub](https://github.com/NUbots/NUbook).

[**NUClear**](https://github.com/Fastcode/NUClear): The software framework which the NUbots codebase is built on. Put simply, it works by triggering callbacks when events occur. Documentation for NUClear is [here](https://nuclear.readthedocs.io/en/latest/index.html).

[**NUClear Binary Stream (NBS)**](https://nubook.nubots.net/system/foundations/build-system#nbs): NUClear logging file format, which is basically concatenated protobuf messages with headers.

[**NUClear Powerplant**](https://nuclear.readthedocs.io/en/latest/components.html#powerplant): The central message system which connects the reactors.

[**NUClear Reactor**](https://nuclear.readthedocs.io/en/latest/components.html#reactors): A NUClear module. Modules in the NUbots codebase are reactors.

[**NUClear Reaction**](https://nuclear.readthedocs.io/en/latest/components.html#reactions): The callbacks which need to run when an event or conditions indicate as such. These manifest as DSL [on statements](https://nuclear.readthedocs.io/en/latest/dsl.html#on-statements).

[**NUClear Roles**](https://github.com/Fastcode/NUClearRoles): The build and messaging system used to manage the construction and messaging of NUClear for the NUbots codebase.

**NUFSR**: A system designed to detect if a NUgus' foot is on the ground, by use of force sensitive resistors (FSRs).

[**NUgan**](https://github.com/NUbots/NUgan): A custom [cycle-gan](https://junyanz.github.io/CycleGAN/) implementation to generate more realistic images for training the vision networks.

[**NUgus**](https://nubook.nubots.net/system/hardware/overview#igus-modifications): NUbots robots as currently used in the RoboCup humanoid league. They are modified [igus Humanoid Open Platform](https://arxiv.org/abs/1809.11110) robots, which we 3D-printed.

[**NUpbr**](https://github.com/NUbots/NUpbr): The software NUbots uses to generate Physically Based Rendered football fields, used to train vision networks.

[**NUsight**](https://github.com/NUbots/NUsight2): NUbots' visualisation software, ran through a browser on an external computer. Source is available [here](https://github.com/NUbots/NUsight).

**NUsense**: A secret project for sensor IO.

**Old Walk Engine**: A previously used walk engine, originally optimised using genetic algorithms.

**Quintic Walk**: Open loop walk engine which uses quintic splines to create trajectories. Created by Bit-Bots, based off code from team Rhoban, then ported to run with NUClear.

**ScriptTuner**: A NUbots program for manually creating motion targets for movement scripts.

**ScriptRunner**: The program which runs movement scripts on the robot, such as `Zombie.yaml` which is used to test servo installations.

[**VisualMesh**](https://arxiv.org/abs/1807.08405): A computer vision technique NUbots uses which overlays a mesh over the image input to more efficiently sample it and look for objects. Source can be found [here](https://github.com/Fastcode/VisualMesh).

## General

[**Affine Transform**](https://en.wikipedia.org/wiki/Affine_transformation): A transform which keeps straight lines straight, and parallel lines parallel, without preserving distances or angles between lines otherwise.

[**Container**](https://cloud.google.com/containers): An isolated environment which decouples the environment an application runs in from the environment it's developed in.

[**Continuous Integration**](https://www.atlassian.com/continuous-delivery/continuous-integration): A development practice whereby frequent updates to a codebase from multiple developers are automated and integrated.

[**Convolution**](https://en.wikipedia.org/wiki/Convolution): A function which transforms two functions into another. [Convolutional Neural Networks](https://en.wikipedia.org/wiki/Convolutional_neural_network) (CNNs) are very commonly used for image processing and computer vision, working by passing filters across the image and combining those outputs in further layers.

[**Deep Learning**](https://en.wikipedia.org/wiki/Deep_learning): Machine learning techniques which utilise artificial neural networks with many layers.

[**Euler Angles**](https://en.wikipedia.org/wiki/Euler_angles): Roll, Pitch, and Yaw - they describe rotation for three orthogonal axes.

[**Extrapolation**](https://en.wikipedia.org/wiki/Extrapolation): Estimation of unknown data points outside the range of observed points.

[**Forward Kinematics**](https://en.wikipedia.org/wiki/Forward_kinematics): Calculation of the end point of a kinematic chain based off the joint angles. The inverse of inverse-kinematics.

[**Inertial Measurement Unit (IMU)**](https://en.wikipedia.org/wiki/Inertial_measurement_unit): A device which measures acceleration with accelerometers and rotation using gyroscopes.

[**Interpolation**](https://en.wikipedia.org/wiki/Interpolation): Estimation of unknown data points within the range of observed points. When used in reference to splines, it means finding a polynomial which satisfies the sets of points, which will be made part of the spline.

[**Inverse Kinematics**](https://en.wikipedia.org/wiki/Inverse_kinematics): Calculation of joint angles for a given position and orientation for the end of the kinematic chain, relative to the start of the kinematic chain. The inverse of forward kinematics.

[**Kalman Filter**](https://en.wikipedia.org/wiki/Kalman_filter): Control theoretic technique for estimating variables by estimating a joint probability distribution over the variables for each timeframe.

[**Kinematics**](https://en.wikipedia.org/wiki/Kinematics): Mathematics which describes motion, which is used to calculate trajectories and servo outputs to control robots.

[**Lambda Expression**](https://en.cppreference.com/w/cpp/language/lambda): A way of defining an anonymous function in C++.

[**Localisation**](https://en.wikipedia.org/wiki/Robot_navigation): The system the robot uses to work out its position relative to its environment and objects within it.

[**Machine Learning**](https://en.wikipedia.org/wiki/Machine_learning): Systems and models which improve their outputs by exposure to inputs.

[**Odometry**](https://en.wikipedia.org/wiki/Odometry): Estimation of change in position and orientation based on sensor inputs.

[**Mahony Filter**](https://nitinjsanket.github.io/tutorials/attitudeest/mahony): A complementary filter which estimates attitude, angle, and orientation using IMU data.

[**Motion Capture (mocap)**](https://en.wikipedia.org/wiki/Motion_capture): The recording of movements of moving objects, which keeps track of specific points on the object as it moves.

[**Spline**](https://en.wikipedia.org/wiki/Spline_(mathematics)): A function defined piecewise by polynomials.

[**Transform**](https://en.wikipedia.org/wiki/Transformation_matrix): Short for transformation. Sometimes, a matrix representation of a transformation.

[**Transformation**](<https://en.wikipedia.org/wiki/Transformation_(function)>): A function which maps a set to itself (sometimes called an operator on that set). Often, it is used in the context of a spatial manipulation - regularly representated as a translation and a rotation.

[**Zero Moment Point (ZMP)**](https://en.wikipedia.org/wiki/Zero_moment_point): With relation to walking motion, it is the point with respect to which the contact force of the foot on the ground produces zero moment in the horizontal direction.

## External

[**Arch**](https://www.archlinux.org/): A lightweight linux distribution which the NUgus robots use as their primary operating system.

[**Bit-Bots**](https://bit-bots.de/): A RoboCup team from the University of Hamburg, which created the quintic walk NUbots uses.

[**Armadillo (arma)**](http://arma.sourceforge.net/): The C++ linear algebra library developed by the CSIRO, previously used by NUbots. Documentation can be found [here](http://arma.sourceforge.net/docs.html).

[**NimbRo**](http://www.nimbro.net/): A RoboCup team from the University of Bonn, Germany.

[**Buildkite**](https://buildkite.com/): A platform for running fast, secure, and scalable continuous integration pipelines, which NUbots runs on machines in the NUbots lab.

[**Clang**](https://clang.llvm.org/): A compiler, frontend, and tooling infrastructure used for C and C++ which is part of the LLVM infrastructure.

[**ClangFormat**](https://clang.llvm.org/docs/ClangFormat.html): A software which formats source code according to a given set of rules.

[**CM740**](<http://support.robotis.com/en/product/robotis-op2/sub_controller(cm-740).htm>): A _Robotis_ sub-controller which gives the commands to the servos.

[**CMake**](https://cmake.org/): An open-source build system generator NUbots uses to configure and build the main codebase.

[**Darwin**](https://robots.ieee.org/robots/darwin/): A _Robotis_ robot platform which was the platform NUbots used before making the NUgus.

[**Docker**](https://www.docker.com/): A software service for developing containers which enable standardised cross-platform development consistency.

[**Dockerhub**](https://hub.docker.com/): A hosting service for docker container images.

[**Dynamixel**](https://www.robotis.us/dynamixel/): The _Robotis_ servo models which NUgus uses.

[**igus**](https://www.igus.com.au/): The creators of the robotics platform NUgus is based on.

[**igus Humanoid Open Platform**](https://arxiv.org/abs/1809.11110): The 3d printed base design which the NUgus is based on.

[**Eigen**](https://eigen.tuxfamily.org/): The C++ linear algebra library NUbots uses. Documentation can be found [here](https://eigen.tuxfamily.org/dox/index.html).

[**Gazebo**](https://gazebosim.org/): A simulation software used which was used to optimise the "OldWalkEngine" with a genetic algorithm.

[**Git**](https://git-scm.com/): A popular version control program used by NUbots.

[**GitHub**](https://github.com/): A source code repository hosting service, which uses a git backend. NUbots uses GitHub to manage the source code.

[**LLVM**](https://llvm.org/): A set of modular compiler and toolchain technologies.

[**NUC**](https://www.intel.com.au/content/www/au/en/products/boards-kits/nuc.html): Intel's portable computing platform which is used in the NUgus robots.

[**Ninja**](https://ninja-build.org/): The build system NUbots uses for the main codebase. It uses the files and configuration generated by CMake.

[**OpenCL**](https://www.khronos.org/opencl/): Open Computing Language is a framework NUbots uses to create programs which can run on graphics hardware.

[**Protobuf**](https://developers.google.com/protocol-buffers): Protocol Buffers (Protobuf) is a method of serializing structured data, created by Google. Protobuf messages are the means of passing messages between NUClear reactors.

[**Rhoban**](https://www.rhoban-project.fr/): The RoboCup team from Bordeaux University, who created the original quintic walk adapted by Bit-Bots, and ported to the NUbots codebase.

[**RoboCup**](https://www.robocup.org/): The robotic football world cup, held each year in different cities around the world.

[**RoboCup Symposium**](https://www.robocup.org/symposium): The research conference which is held after RoboCup each year, featuring research from the teams competing.

[**Robotis**](https://www.robotis.us/): The Korean robotics manufacturer which makes the servos that NUgus uses.

[**Simulink**](https://www.mathworks.com/products/simulink.html): A MATLAB-based graphical simulation software.

[**Simscape**](https://www.mathworks.com/products/simscape.html): A MATLAB-based simulation tool for creating models of physical systems in the Simulink environment.

[**YAML**](https://yaml.org/): A markup/serialisation language NUbots uses for module configurations.
